# Accessibility Requirements

> **Status**: In Design
> **Author**: user + ux-designer
> **Last Updated**: 2026-06-27

---

## Accessibility Tier

**Target**: WCAG-AA 基线 + 游戏特定扩展

本作是 Web 游戏，需在浏览器中运行。遵循 W3C WCAG 2.1 AA 标准作为基线，并针对 Survivor-like 游戏特性扩展。

---

## Core Requirements

### 1. 视觉 (Visual)

#### 1.1 颜色无障碍

| 要求 | 标准 | 实现 |
|------|------|------|
| 颜色对比度 | WCAG AA: 文本 ≥4.5:1, 大文本 ≥3:1 | 墨黑 `#1A1D22` vs 宣纸白 `#F0E8D8` = 12.5:1 ✅ |
| 颜色不作为唯一信息通道 | 所有关键信息有形状/位置/音效备份 | 色盲安全矩阵 (art-bible Section 4.6) |
| 色盲支持 | Deuteranopia, Protanopia, Tritanopia | 元素色有形状+音效备份 |

**色盲安全矩阵** (art-bible Section 4.6 锁定):

| 颜色对 | 风险 | 备份通道 |
|--------|------|----------|
| 松绿 ↔ 朱砂红 | Deuteranopia/Protanopia | 叶形轮廓 + 脉冲径向 + 音效 |
| 金 ↔ 松绿 | Deuteranopia | 径向爆发形状 + 叶形 |
| 紫霞 ↔ 云青 | Tritanopia | 灵魂粒子轨迹 + 背景 |
| 朱砂红 ↔ 墨黑 | 低对比度 | 1帧纸白闪光 + 震屏 + 音效 |

#### 1.2 文本可读性

| 要求 | 标准 |
|------|------|
| 最小字体 | 14px (Caption), 18px (Body) |
| 字体选择 | 等宽数字 (JetBrains Mono), 楷体 (Noto Serif CJK SC) |
| 行高 | ≥1.4 (正文), ≥1.0 (数字) |

#### 1.3 动画

| 要求 | 实现 |
|------|------|
| 减少动画选项 | 设置中提供 "减少动画" 开关 |
| 闪烁频率 | <3次/秒 (WCAG 2.3.1) |
| 运动 sickness | 突破动画可跳过 (设置) |

---

### 2. 输入 (Input)

#### 2.1 键盘导航

| 要求 | 实现 |
|------|------|
| 所有交互元素可达 | Tab 顺序遍历所有按钮/菜单项 |
| 焦点指示器 | 朱砂红外框 2px, 始终可见 |
| 焦点陷阱 | 模态弹窗内 Tab 循环 |
| 快捷键 | ESC 关闭模态, Enter 确认 |

#### 2.2 触屏

| 要求 | 标准 |
|------|------|
| 最小触控目标 | 44×44px (W3C / Apple HIG) |
| 间距 | ≥8px 相邻可点击元素 |
| 虚拟摇杆 | 80px + 16px 超调 |
| 悬停替代 | 无 hover 交互, 长按替代 |

#### 2.3 手柄

| 要求 | 实现 |
|------|------|
| 导航 | D-pad/摇杆移动焦点 |
| 确认/取消 | A 确认, B 取消 |
| 暂停 | Start 按钮 |

---

### 3. 认知 (Cognitive)

#### 3.1 信息层次

| 要求 | 实现 |
|------|------|
| 关键信息突出 | 灵气条/技能槽/BOSS 血条始终可见 |
| 次要信息渐进 | HUD 透明度策略 (idle α=0.35, combat α=1.0) |
| 一致性 | 所有屏幕使用相同交互模式库 |

#### 3.2 错误预防

| 要求 | 实现 |
|------|------|
| 不可逆操作确认 | 功法装备前确认 (不可卸下) |
| 清晰反馈 | 每次操作有视觉+音效反馈 |
| 恢复路径 | 死亡后"重来"按钮居中 |

---

### 4. 听觉 (Auditory)

#### 4.1 音效

| 要求 | 实现 |
|------|------|
| 音效不作为唯一信息通道 | 所有音效有视觉备份 |
| 音量控制 | 设置中独立调节 BGM/SFX |
| 静音可用 | 游戏可在完全静音下正常进行 |

## Game-Specific Accessibility

### Survivor-like 特定

| 挑战 | 解决方案 |
|------|----------|
| 150+ 实体可读性 | art-bible 强制留白圈 + 飞白/泼彩层级 |
| 快节奏战斗 | 自动释放功法, 降低操作门槛 |
| 长时间游戏 (30-40min) | HUD 透明度策略减少视觉疲劳 |
| 境界突破仪式感 | 1.2s 动画, 可通过设置缩短 |

### Web 平台特定

| 挑战 | 解决方案 |
|------|----------|
| 浏览器兼容性 | Compatibility 渲染器 (OpenGL 3.3 / WebGL2) |
| 加载时间 | 50MB 限制, 首屏预加载 |
| 键盘冲突 | 避免浏览器快捷键冲突 (如 F5) |

## Testing Checklist

### 色盲测试

- [ ] Deuteranopia: 松绿/朱砂红 可区分
- [ ] Protanopia: 所有元素色可区分
- [ ] Tritanopia: 紫霞/云青 可区分
- [ ] 灰度模式: 所有关键信息可读

### 键盘测试

- [ ] Tab 遍历所有交互元素
- [ ] 焦点指示器始终可见
- [ ] ESC 关闭所有模态
- [ ] Enter 确认所有按钮

### 触屏测试

- [ ] 所有按钮 ≥44×44px
- [ ] 虚拟摇杆响应正确
- [ ] 无 hover 依赖交互

### 屏幕阅读器测试

- [ ] 模态弹窗有 aria-modal
- [ ] 按钮有可读标签
- [ ] 状态变化有 aria-live 通知
